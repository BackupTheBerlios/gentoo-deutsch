Das Gentoo Linux alternative Installations Methoden HOWTO

Inhalt:

1.Über dieses Dokument

Wenn die Standard Boot-von-CD Installation für Sie nicht funktioniert (oder Sie sie einfach nicht mögen), ist nun Hilfe verfügbar. Dieses Dokument soll eine Sammlung verschiedener alternativer Installations Methoden bieten. Sollten Sie eine alternative Installations Methode kennen, von der Sie meinen, daß sie auch für andere hilfreich sein könnte, zögern Sie bitte nicht mir zu schreiben (Englisch), bzw. sich an das Team von Gentoo.de zu wenden.

2.Netboot Installation

Voraussetzungen 

Die Voraussetzungen für eine Netboot Installation sind ein Ziel-Computer, wo ein tftp-Server läuft, und ein Computer, der sich selber via netboot entweder vom BIOS oder von Diskette aus, wo GRUB oder ein anderer Netzwerk-Bootloader installiert ist, vom Netzwerk bootet. Ein dhcp-Server ist unter Umständen ebenfalls nötig. Natürlich brauchen wir auch das aktuellste ISO, das unter http://www.ibiblio.org/gentoo/snapshots/build/ gefunden werden kann.

Überblick 

Als erstes muss die Netboot-Maschine eine IP Adresse beziehen, bevor es das Images aus dem Netzwerk ziehen kann. Es gibt mehrere Möglichkeiten, um an eine IP-Adresse zu kommen, und jede wird genügen. Persönlich bevorzuge ich GRUB für alles, aber wenn Ihr Computer bereits das Booten vom Netzwerk unterstützt, ist GRUB nicht unbedingt notwendig. Auch wenn es vielleicht einfach ist GRUBs ifconfig Kommando zu verwenden, und keinen bootp oder dhcp Server zu installieren.

Nachdem Ihr Computer eine IP Adresse bezogen hat folgt der nächste logische Schritt. Wie müssen herausfinden, was gebootet werden soll und wo es aufbewahrt wird. Nochmals wäre es natürlich einfacher mit GRUB Kommandos zu arbeiten, anstatt einen BOOTP oder DHCP Server aufzusetzen. Außerdem müssen Sie festlegen wie eine Initrd erstellt werden soll und wie der Kernel diese als ROOT-Dateisystem verwenden soll.

Wenn Ihr Kernel geladen und das ROOT-Dateisystem gemountet wurde, können Sie mit einer normalen Installation fortfahren. Das Installations Image kann entweder von CD geladen oder via tftp vom Netzwerk bezogen werden.

GRUB verwenden 

Damit GRUB für das Netzwerkbooten genutzt werden kann, muss es erst mit Unterstützung für Ihre Netzwerkkarte kompiliert werden. Dabei spielt es keine Rolle, ob Sie GRUB auf eine Diskette oder Festplatte installieren wollen. Sollte Ihr Installationsziel bereits über ein GRUB mit Netzwerkunterstützung verfügen, können Sie beim nächsten Schritt fortfahren. GRUB kann von ftp://alpha.gnu.org/gnu/grub/ heruntergeladen werden.

Ein Konfigurations-Beispiel mit Tulip Unterstützung, die Netzwerkkarte in meinem Rechner:

Beispiel 1: Manuelle GRUB Installation

# ./configure --enable-tulip --prefix=/usr
# make && make install

Falls Sie sich bereits in Gentoo befinden sollten und GRUB mit Netzwerkunterstützung durch Gentoo-Tools installieren möchten, sollten Sie die folgenden Schritte genau befolgen. Ein Beispiel, um ein Ebuild zur Installation von GRUB mit Tulip Unterstützung zu nutzen:

Beispiel 2: Installation und Konfiguration von GRUB unter Gentoo Linux

# ebuild /usr/portage/sys-apps/grub/grub-0.91.ebuild clean fetch unpack
# cd /var/tmp/portage/grub-0.91/work/grub-0.91/
# ./configure --prefix=/usr --sbindir=/sbin --mandir=/usr/share/man \ 
> --infodir=/usr/share/info --enable-tulip
# make
# touch /var/tmp/portage/grub-0.91/.compiled
# cd /usr/portage/
# ebuild sys-apps/grub/grub-0.91.ebuild install merge

Nachdem wir die GRUB Shell nun installiert haben, müssen wir GRUB noch in einen Bootsektor installieren. Auch wenn Sie GRUB direkt in den Bootsektor der Festplatte Ihres Installations Systems installieren könnten, gehen wir hier davon aus, dass die Installation auf eine Diskette erfolgt. Dafür gibt es zwei Möglichkeiten. Sie können zum einen direkt die GRUB Shell oder ein mitgeliefertes Script, grub-install genannt, benutzen. Sie sollten grub-install zur Installation auf einer Diskette den Vorzug geben.

Beispiel 3: GRUB Installations Beispiel

# mkfs.ext2 /dev/fd0
# mount /dev/fd0 /mnt/floppy/
# grub-install --root-directory=/mnt/floppy/ '(fd0)'
# umount /mnt/floppy/

grub-install funktioniert nicht immer ... und ist nicht immer der beste Weg zur Installation. Da die GRUB Shell genauso arbeitet, wie GRUB beim Booten, könnte es wünschenswert sein die GRUB Shell zu benutzten. Hier ist ein Beispiel zur Verwendung der GRUB Shell zur Installation von GRUB auf einer Diskette:

Beispiel 4: Benutzung der GRUB Shell

# mkfs.ext2 /dev/fd0
# mount /dev/fd0 /mnt/floppy/
# mkdir -p /mnt/floppy/boot/grub/
# cp -v /usr/share/grub/i386-pc/* /mnt/floppy/boot/grub/
# grub
grub> root (fd0)
grub> setup (fd0)
grub> quit
# umount /mnt/floppy/

Da wir nun eine bootfähige GRUB Diskette haben, müssen wir nun einen tftp Server (ich schlage Netkit's TFTP Server vor) für das Laden des Kernels und der Initrd aufsetzen. Sollten Sie Inetd nutzen, benötigen Sie folgende Zeile in Ihrer /etc/inetd.conf:

Beispiel 5: /etc/inetd.conf

tftp	dgram	udp	wait	nobody	/usr/sbin/tcpd	in.tftpd

Um den NEtkit tftp Server unter Gentoo Linux zu installieren, benutzen Sie "emerge net-misc/netkit-tftp".

Anmerkung: Es gibt ein Ebuild für Xinetd ... sollten Sie dieses bevorzugen, können Sie es natürlich ebenfalls verwenden. Auf jeden Fall benutzt ich es nicht, daher weiß ich auch nicht, wie man es mit tftp einrichtet. Sollten Sie es benutzten, wäre es sehr freundlichen, wenn Sie mir Details mailen könnten. Dann könnte ich dies in diesem Howto integrieren.

Da wir nun einen tftp Server am Laufen haben, brauchen wir nur noch einen Kernel und eine ROOT-Initrd. Sie können einen eigenen Kernel kompilieren, aber achten Sie darauf, dass dieser alle wichtigen Dinge, die für das funktionieren von Gentoo Linux nötig sind (wie devfs und initrd fürs Netbooting), enthält. Die ROOT Initrd stellt die Datei rescue.gz vom Gentoo ISO.

Wichtig: Zum Mounten einer ISO-Datei ohne sie zu brennen, wird die Unterstützung für das Loopback-Dateisystem benötigt.

Beispiel 6

# mkdir /tftpboot
# mount -o loop /path/to/build-ix86-1.0_rc6-r14.iso /mnt/cdrom/
# cp /mnt/cdrom/isolinux/kernel /mnt/cdrom/isolinux/rescue.gz /tftpboot
# chmod 644 /tftpboot/*
# umount /mnt/cdrom/

Booten Sie nun den Rechner auf dem Gentoo installiert werden soll mit Ihrer unglaublich nützlichen GRUB Diskette. Nachdem Sie gebootet haben, müssen Sie noch den Weg festlegen, wie die IP Adresse bezogen werden soll. Nun legen Sie noch fest woher der Kernel und seine Initrd bezogen werden sollen.

Beispiel 7

grub> ifconfig --address=192.168.0.10 --server=192.168.0.2
grub> root (nd)
grub> kernel /tftpboot/kernel devfs=nomount vga=normal load_ramdisk=1 
      prompt_ramdisk=0 ramdisk_size=18580 root=/dev/ram0 rw (alles in eine Zeile)
grub> initrd /tftpboot/rescue.gz
grub> boot

Anmerkung: Sie können auch bootp und dhcp verwenden, um Ihre IP via GRUB zu konfigurieren. Verwenden Sie die bootp und dcpd Kommandos.

Jetzt wo Ihre Maschine gebootet ist, können Sie wie immer installieren. Lesen Sie dazu im HOWTO zur Installation von CD weiter.
